# Help, Text, and UX

The core does not define which package or help component each project uses. That must come from the local profile.

## Generic Rule

- Long help should be centralized and reusable.
- The main surface should show only the minimum context needed to perform the task.
- Tutorials, long rules, and examples should live in contextual help.
- Visible text should be clear for end users.
- Do not expose tables, columns, packages, internal IDs, or raw exceptions.

## When Working in a Project

Read `.oracle-apex-ai/project-profile.md` to discover:

- where help content lives;
- how help opens;
- which component to use;
- which package to call;
- which business terms to use;
- which technical terms are forbidden.
